# Creating an Application

## The Component Entry Point

Every Core application starts by loading a **root component** using the `component()` function:

```js
import { component } from "core/handlebar";

const App = await component("App.html");
```

---

## The Root Component

The value returned by `component()` is a fully compiled component that can be mounted into the DOM.

Most real applications are composed of multiple reusable components organized in a tree-like structure:

```text
App (root component)
├─ Header
├─ Sidebar
├─ TodoList
│  ├─ TodoItem
└─ Footer
```

In later sections, we will explore how components are composed, reused, and nested within each other.

---

## Mounting the Application

A Core application does not render anything until it is explicitly mounted using `mount()`:

```html
<div id="app"></div>
```

```js
import { component } from "core/handlebar";
import { mount } from "core";

const App = await component("App.html");
mount(App, "#app");
```

The application is rendered inside the target container. The container itself is not considered part of the application.

---

## Application Scope

Each mounted Core application instance is isolated in its own scope:

- State is scoped to the component tree
- Signals are tracked per instance
- Effects are cleaned up when the component is unmounted

This allows multiple Core applications to coexist on the same page without interference.

---

## Multiple Application Instances

You are not limited to a single Core application per page.

```js
import { component } from "core/handlebar";
import { mount } from "core";

const App1 = await component("App1.html");
const App2 = await component("App2.html");

mount(App1, "#container-1");
mount(App2, "#container-2");
```

Each instance runs independently with its own reactive graph and lifecycle.
