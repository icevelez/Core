# What's Next?

Congratulations! You've reached the end of the Essentials guide.

At this point, you should have a solid understanding of Core's fundamental concepts, including components, reactivity, template syntax, lifecycle hooks, context, and rendering.

The rest of the documentation explores Core in greater depth and focuses on specific areas of the framework. Depending on what you're interested in, there are several paths you can take next.

---

## Components In Depth

If you want to learn more about how Core's component system works, continue with the Components In Depth guide.

Topics include:

- Async Components
- Component Import Resolution
- Component Caching
- Component Events
- Circular Dependencies

Unlike traditional JavaScript modules, Core components are loaded and resolved through the component runtime. This section explains how the system works and the tradeoffs behind its design.

---

## Included Libraries

Core ships with a collection of optional libraries that solve common application needs while remaining framework-friendly.

### Client Side Router

Build client-side applications using Core's router.

### Manager

Manager is Core's state management library inspired by Flux and Redux.

### Valid Type

Valid Type provides runtime validation and type checking utilities.

### Util

Utility contains a collection of helper functions used throughout Core applications.
