# Component Lifecycle Hooks

Components often need to perform work when they are created and clean up resources when they are removed.

Core provides lifecycle hooks that allow you to run code at specific stages of a component's lifetime.

```html
<script>
    import { on_mount, on_destroy } from "core";

    export default function() {

        on_mount(() => {
            console.log("Mounted");

            // cleanup function
            return () => {
                console.log("Mount cleanup");
            };
        });

        on_destroy(() => {
            console.log("Destroyed");
        });
    }
</script>
```

Lifecycle hooks are useful for:

* Fetching data
* Setting up subscriptions
* Starting timers
* Attaching event listeners
* Initializing third-party libraries
* Cleaning up resources

---

## Component Lifetime

A component generally moves through two lifecycle stages:

```text
Component Created
        ↓
      Mount
        ↓
   Component Active
        ↓
     Destroy
```

Hooks allow you to run code during the mount and destroy phases.

---

## on_mount

The `on_mount()` hook runs after the component has been created and mounted into the DOM.

```js
on_mount(() => {
    console.log("Component mounted");
});
```

This is commonly used for:

* Fetching data
* Starting timers
* Accessing DOM elements
* Initializing libraries

### Example

```js
on_mount(() => {
    const timer = setInterval(() => {
        console.log("Tick");
    }, 1000);
});
```

---

## Mount Cleanup

`on_mount()` may optionally return a cleanup function.

The cleanup function is executed when the component is destroyed.

```js
on_mount(() => {

    const timer = setInterval(() => {
        console.log("Tick");
    }, 1000);

    return () => {
        clearInterval(timer);
    };
});
```

This pattern is useful because setup and cleanup logic remain together.

```text
Setup
  ↓
Cleanup
```

---

## on_destroy

The `on_destroy()` hook runs immediately before the component is removed.

```js
on_destroy(() => {
    console.log("Component destroyed");
});
```

This hook is commonly used for:

* Closing connections
* Removing subscriptions
* Releasing resources
* Manual cleanup

### Example

```js
const socket = create_socket();

on_destroy(() => {
    socket.close();
});
```

---

## Using Both Hooks

It is common to use both hooks together.

```js
on_mount(() => {
    console.log("Mounted");
});

on_destroy(() => {
    console.log("Destroyed");
});
```

Execution order:

```text
Component Created
        ↓
on_mount
        ↓
Component Active
        ↓
on_destroy
```

---

## Multiple Hooks

A component may register multiple lifecycle hooks.

```js
on_mount(() => {
    console.log("Mount A");
});

on_mount(() => {
    console.log("Mount B");
});
```

```js
on_destroy(() => {
    console.log("Destroy A");
});

on_destroy(() => {
    console.log("Destroy B");
});
```

Each hook will execute in the order it was registered.

---

## Cleaning Up Event Listeners

Lifecycle hooks are commonly used when interacting with browser APIs.

```js
on_mount(() => {

    const handleResize = () => {
        console.log(window.innerWidth);
    };

    window.addEventListener("resize", handleResize);

    return () => {
        window.removeEventListener("resize", handleResize);
    };
});
```

This ensures event listeners do not remain after the component has been removed.

---

## Fetching Data

Data fetching is often performed during mount.

```js
on_mount(async () => {

    const response = await fetch("/api/users");

    const users = await response.json();

    console.log(users);
});
```

The request begins when the component becomes active.

---

## Relationship to Effects

Lifecycle hooks differ from effects.

### Lifecycle Hooks

```js
on_mount(() => {
    console.log("Runs once");
});
```

* Run at specific lifecycle stages
* Do not react to signal changes
* Typically execute once

### Effects

```js
effect(() => {
    console.log(count());
});
```

* React to signal updates
* Re-run automatically
* Used for reactive behavior

Use lifecycle hooks for setup and cleanup.

Use effects for reactive updates.

---

## Key Concepts

* `on_mount()` runs when a component is mounted.
* `on_destroy()` runs when a component is destroyed.
* `on_mount()` may return a cleanup function.
* Lifecycle hooks are useful for setup and teardown logic.
* Effects and lifecycle hooks serve different purposes.
* Proper cleanup helps prevent memory leaks and resource leaks.
