# Component Styling

## Overview

Core allows components to define their own styles directly inside a `<style>` block.

```html
<style>
    h1 {
        color: red;
    }
</style>

<h1>Hello World</h1>
```

Similar to Vue, styles declared inside a Core component are global by default and require the `scoped` attribute to tell Core that this style is scoped to the component

---

## Basic Usage

Simply place a `<style scoped>` tag anywhere at the top-level in your component:

```html
<style scoped>
    h1 {
        color: red;
    }
    
    p {
        color: gray;
    }
</style>

<h1>John Doe</h1>
<p>Administrator</p>
```

Internally, Core assigns a unique scope identifier to the component and applies it to both:

* The rendered DOM elements
* The component's CSS selectors

Conceptually:

```html
<h1 scope-abc>Hello World</h1>
```

```css
h1[scope-abc] {
    color: red;
}
```

This ensures the style only affects elements belonging to that component.

> [!NOTE]
> The actual scope identifier is generated automatically and is not intended to be referenced directly.

---

## Similarities to Vue and Svelte

If you have used Vue's `<style scoped>` or Svelte's component styles, the behavior will feel familiar.

### Vue

```vue
<style scoped>
    h1 {
        color: red;
    }
</style>
```

### Svelte

```svelte
<style>
    h1 {
        color: red;
    }
</style>
```

### Core

```html
<style scoped>
    h1 {
        color: red;
    }
</style>
```

In all three cases, the goal is the same:

> Keep component styles isolated from the rest of the application.

The primary difference is that Core performs style scoping at runtime rather than during a build step.

---

## Using Classes

Classes work exactly as they do in normal CSS.

```html
<style scoped>
    .card {
        border: 1px solid #ccc;
        padding: 1rem;
    }
</style>

<div class="card">
    User Profile
</div>
```

The generated scoped CSS remains isolated to the component.

---

## Using IDs

IDs are also automatically scoped.

```html
<style scoped>
    #header {
        color: blue;
    }
</style>

<h1 id="header">
    Dashboard
</h1>
```

Although IDs are supported, classes are generally recommended for reusable styling.

---

## Complex Selectors

Most common selectors work normally.

```html
<style scoped>
    .card h1 {
        margin-bottom: 1rem;
    }
    
    .card > p {
        color: gray;
    }
    
    button:hover {
        background: black;
    }
</style>
```

Core automatically applies scoping to each selector while preserving the original selector behavior.

---

## Media Queries

Media queries are fully supported.

```html
<style scoped>
    .card {
        padding: 2rem;
    }
    
    @media (max-width: 768px) {
        .card {
            padding: 1rem;
        }
    }
</style>
```

Scoped selectors remain scoped even inside media queries.

---

## Multiple Style Blocks

A component may contain multiple `<style>` blocks.

```html
<style scoped>
    h1 {
        color: red;
    }
</style>

<style scoped>
    p {
        color: gray;
    }
</style>
```

All styles are collected and scoped as part of the component compilation process.

---

## Styling Child Components

Component CSS only applies to elements rendered by the current component.

Consider:

```html
<UserCard>
    <Avatar />
</UserCard>
```

```html
<style scoped>
    img {
        border-radius: 50%;
    }
</style>
```

The style affects only images rendered by `UserCard`.

Images rendered inside `Avatar` remain unaffected.

This behavior differs from techniques such as nested CSS or `@scope`, which operate on the final DOM tree and may affect descendant components.

---

## Global Styles

Sometimes styles should apply application-wide rather than to a single component.

In these situations, place the styles in a global stylesheet:

```html
<link rel="stylesheet" href="./app.css">
```

or:

```html
<!-- index.html -->
<style>
    body {
        margin: 0;
    }
</style>
```

within your application entry point.

Component CSS should generally be used for styles that belong to a specific component.

---

## How It Works

When a component is loaded:

1. Core extracts all `<style>` blocks
2. A unique scope identifier is generated
3. CSS selectors are rewritten to include the scope identifier
4. Component elements receive the same scope identifier
5. The resulting stylesheet is injected into the document

Conceptually:

```text
Component
    ↓
Extract Styles
    ↓
Generate Scope ID
    ↓
Rewrite Selectors
    ↓
Inject Styles
    ↓
Render Component
```

This process happens automatically and requires no configuration.

---

## Summary

Component CSS allows styles to live alongside the markup they belong to while remaining isolated from the rest of the application.

```html
<style scoped>
h1 {
    color: red;
}
</style>

<h1>Hello World</h1>
```

Core automatically scopes these styles at runtime, providing a development experience similar to Vue's `<style scoped>` and Svelte's component styles
