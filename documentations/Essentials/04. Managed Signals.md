# Managed Signals

Managed Signals are an advanced state primitive built on top of Core's signal system.

They combine:

* Reactive state
* Error handling
* Loading state
* Validation
* Transformation
* Action-based state updates

into a single abstraction.

Managed Signals are useful when state requires more than simple reads and writes.

---

## Creating a Managed Signal

```js
import { managed_signal } from "core";
```

```js
const [
    value,
    error,
    pending,
    controller
] = managed_signal(initial_value, config);
```

Returns:

1. State getter
2. Error getter
3. Pending getter
4. Bound actions or refined setter

---

# Refinement Mode

When the second argument is a function, the managed signal acts as a refined signal.

```js
const [
    username,
    error,
    pending,
    setUsername
] = managed_signal(
    "",
    value => value.trim()
);
```

Every update passes through the refinement function before being stored.

---

## Transforming Values

```js
const [
    email,
    error,
    pending,
    setEmail
] = managed_signal(
    "",
    value => value.trim().toLowerCase()
);

setEmail("  JOHN@EXAMPLE.COM ");

console.log(email());
```

Output:

```text
john@example.com
```

This allows values to be normalized before entering state.

---

## Validating Values

The refinement function may throw an error.

```js
const [
    age,
    error,
    pending,
    setAge
] = managed_signal(
    18,
    value => {
        if (value < 0) {
            throw new Error(
                "Age cannot be negative"
            );
        }

        return value;
    }
);
```

```js
setAge(-5);

console.log(error()?.message);
```

Output:

```text
Age cannot be negative
```

When validation fails:

* The state is not updated
* The error signal is updated

---

## Validation and Transformation

Validation and transformation can be combined.

```js
const [
    email,
    error,
    pending,
    setEmail
] = managed_signal(
    "",
    value => {
        const normalized =
            value.trim().toLowerCase();

        if (!normalized.includes("@")) {
            throw new Error(
                "Invalid email address"
            );
        }

        return normalized;
    }
);
```

```js
setEmail(" JOHN@EXAMPLE.COM ");

console.log(email());
```

Output:

```text
john@example.com
```

---

## Functional Updates

Refinement mode supports updater functions.

```js
const [
    count,
    error,
    pending,
    setCount
] = managed_signal(
    0,
    value => Math.max(0, value)
);
```

```js
setCount(current => current - 10);
```

The updater receives the current state before passing through the refinement pipeline.

---

## Async Refinement

The refinement function may return a Promise.

```js
const [
    username,
    error,
    pending,
    setUsername
] = managed_signal(
    "",
    async value => {
        const exists =
            await check_username(value);

        if (exists) {
            throw new Error(
                "Username already exists"
            );
        }

        return value;
    }
);
```

```js
await setUsername("john");
```

During execution:

```js
pending();
```

returns:

```text
true
```

When the Promise resolves:

```js
pending();
```

returns:

```text
false
```

---

# Action Mode

When the second argument is an object, the managed signal behaves like a managed state container.

```js
const [
    count,
    error,
    pending,
    actions
] = managed_signal(
    0,
    {
        increment(state) {
            return state + 1;
        },

        decrement(state) {
            return state - 1;
        }
    }
);
```

---

## Calling Actions

```js
actions.increment();

console.log(count());
```

Output:

```text
1
```

Actions receive the current state as their first argument.

```js
increment(state) {
    return state + 1;
}
```

The returned value becomes the next state.

---

## Actions With Parameters

Actions may accept additional arguments.

```js
const [
    count,
    error,
    pending,
    actions
] = managed_signal(
    0,
    {
        add(state, amount) {
            return state + amount;
        }
    }
);
```

```js
actions.add(5);

console.log(count());
```

Output:

```text
5
```

---

## Async Actions

Actions may return Promises.

```js
const [
    user,
    error,
    pending,
    actions
] = managed_signal(
    null,
    {
        async load() {
            const response =
                await fetch("/api/user");

            return response.json();
        }
    }
);
```

```js
actions.load();
```

While the action is running:

```js
pending();
```

returns:

```text
true
```

When the Promise resolves:

```js
pending();
```

returns:

```text
false
```

and the state is updated automatically.

---

## Error Handling

Errors thrown by actions are automatically captured.

```js
const [
    value,
    error,
    pending,
    actions
] = managed_signal(
    0,
    {
        crash() {
            throw new Error(
                "Something went wrong"
            );
        }
    }
);
```

```js
actions.crash();

console.log(error()?.message);
```

Output:

```text
Something went wrong
```

---

# Reactive Error State

Errors are reactive signals.

They can be displayed directly in templates.

```html
<input
    value="{{ username() }}"
    on:input="event => setUsername(event.target.value)"
>

{{#if error()}}
    <p class="error">
        {{ error().message }}
    </p>
{{/if}}
```

Whenever validation fails, the UI updates automatically.

---

# Reactive Pending State

Pending state is also reactive.

```html
<button
    :disabled="pending()"
    on:click="actions.load"
>
    Load User
</button>

{{#if pending()}}
    <p>Loading...</p>
{{/if}}
```

This makes it easy to build loading indicators and disable interactions while asynchronous operations are running.

---

# When To Use Managed Signals

Managed Signals are useful when state requires additional rules or behavior beyond a standard signal.

Common use cases include:

* Form validation
* Input normalization
* Sanitization
* Async validation
* API requests
* Action-based state management
* Business rule enforcement

Examples:

```js
managed_signal("", validate_email);
managed_signal("", sanitize_html);
managed_signal(null, user_actions);
managed_signal([], todo_actions);
```
