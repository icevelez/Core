# What's Next?

Congratulations! You've reached the end of the Essentials guide.

At this point, you should have a solid understanding of Core's fundamental concepts, including components, reactivity, template syntax, lifecycle hooks, context, and rendering.

The rest of the documentation explores Core in greater depth and focuses on specific areas of the framework. Depending on what you're interested in, there are several paths you can take next.

---

## Components In Depth

If you want to learn more about how Core's component system works, continue with the Components In Depth guide.

Topics include:

- Component Import Resolution
- Component Caching
- Component Events
- Circular Dependencies

Unlike traditional JavaScript modules, Core components are loaded and resolved through the component runtime. This section explains how the system works and the tradeoffs behind its design.

## Included Libraries

Core ships with a collection of optional libraries that solve common application needs while remaining framework-friendly.

### Client Side Router

Build client-side applications using Core's router.

### Manager

Manager is Core's state management library inspired by Flux and Redux.

### Valid Type

Valid Type provides runtime validation and type checking utilities.

### Util

Utility contains a collection of helper functions used throughout Core applications.

## Architecture & Design Insights

If you're interested in the ideas that shaped Core, these guides explore the reasoning behind the framework's architecture and how it evolved over time.

### SFC Insight

Learn how Core's Single File Component architecture came to exist and how Core achieves a Vue- and Svelte-like authoring experience entirely in the browser. The guide covers the evolution from class-based components, the introduction of lexical-scope rendering, and the realization that ultimately led to Core's modern component model. 

Topics include:

* The history of Core's component system
* JSC vs SFC components
* Runtime module generation using Blob URLs
* Lexical scope injection
* Why Core v0.5.0 adopted SFCs exclusively
* Similarities to React Hooks and modern compiler output

### Reactivity and Performance

A deeper exploration of how modern JavaScript frameworks approach performance.

This guide breaks framework architecture into four separate concerns:

1. Change Detection
2. Scheduling Granularity
3. DOM Update Granularity
4. DOM Update Strategy 

It compares different framework and explains why statements such as:

> "Signals are faster"

or

> "Virtual DOM is slower"

are often oversimplifications. 

This guide is recommended for developers who want a deeper understanding of frontend rendering architecture.

---

### Core Philosophy

Several guides throughout the documentation explore the ideas that make Core different from most modern frameworks:

* Runtime compilation
* Browser-first development
* Components as runtime resources
* Progressive optimization through Core Assist
* Runtime-extensible applications
* Build-tool optional workflows

These documents focus less on APIs and more on the design philosophy behind the framework.
