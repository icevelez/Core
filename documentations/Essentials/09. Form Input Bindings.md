# Form Input Bindings

Form bindings in Core provide a simple way to synchronize user input with reactive state.

Core supports multiple approaches for handling form state:

- Attribute bindings (`:value`)
- Event bindings (`on:event`)
- Actions (`use:bind`) for two-way binding

These tools can be used individually or combined depending on the level of control required.

---

## Attribute Binding

The `:` prefix binds a DOM attribute to a JavaScript expression.

```html
<input :value="name" />
```

This creates a one-way binding from state → UI.

To handle updates, you can combine it with an event listener:

```html
<input
    :value="name()"
    on:input="(e) => setName(e.target.value)"
/>
```

---

## Event Binding

Core uses `on:` for event listeners.

```html
<button on:click="(e) => some_event(e)">
    Click me
</button>
```

Events receive the native DOM event object.

This is a direct and explicit way to handle user interaction.

---

## Two-Way Binding with `use:bind`

For form inputs, Core provides a convenient `use:bind` action that connects DOM values directly to reactive state.

```html
<input type="text" use:bind="['value', name, setName]" />
```

```js
['attribute', getter, setter]
```

Where:

- **attribute** → DOM property to bind (e.g. `"value"`, `"checked"`)
- **getter** → reactive getter function
- **setter** → reactive setter function

- Reads the current signal value into the input
- Updates the signal when the input changes

## Supported Input Types

### Text Input

```html
<input type="text" use:bind="['value', name, setName]" />
```

Uses the `input` event internally.

---

### Checkbox

```html
<input type="checkbox" use:bind="['checked', show, setShow]" />
```

Checkbox bindings use the `change` event and bind to the `checked` property.

---

### Date Input

```html
<input type="date" use:bind="['value', date, setDate]" />
```

Updates occur on `change`.

---

### Select Dropdown

```html
<select use:bind="['value', chosen, setChosen]">
    <option value="a">A</option>
    <option value="b">B</option>
</select>
```

Select elements synchronize on `change` events.

---

## How `use:bind` Works

Internally, `use:bind` attaches:

- A DOM listener (`input` or `change`)
- A value synchronizer
- A cleanup handler

Conceptually:

```
DOM Element
   ↔
Signal Getter/Setter
   ↔
Reactive System
```

It ensures both directions stay in sync:

- UI → State (user input)
- State → UI (programmatic updates)
